## Project Overview

DPDK (Data Plane Development Kit) is a high-performance packet processing framework written in C. It supports multiple processor architectures and operating systems (FreeBSD and Linux). The project uses the BSD-3-Clause license for core libraries/drivers and GPL-2.0 for kernel components.

## Build System

DPDK uses Meson as its primary build system:

### Building DPDK
```bash
# Standard build process
meson setup builddir
ninja -C builddir

# Build with specific configuration
meson setup builddir -Ddeveloper_mode=true
ninja -C builddir

# Install
ninja -C builddir install
```

### Testing
```bash
# Run all tests from build directory
meson test -C builddir

# Run specific test suites
meson test -C builddir --suite fast-tests
```

### Development Tools
The `devtools/` directory contains various scripts for development:
- `devtools/checkpatches.sh` - Check patch formatting and style
- `devtools/check-abi.sh` - Verify ABI compatibility
- `devtools/test-meson-builds.sh` - Test different build configurations

## Architecture

### Core Components
- `lib/` - Core DPDK libraries (packet processing, memory management, etc.)
- `drivers/` - Hardware-specific drivers organized by type (net, crypto, compress, etc.)
- `app/` - Applications and utilities (testpmd, proc-info, etc.)
- `examples/` - Sample applications demonstrating DPDK usage
- `kernel/` - Kernel modules for Linux and FreeBSD
- `usertools/` - Python utilities for device binding and system setup

### Key Libraries
- `lib/eal/` - Environment Abstraction Layer (core initialization)
- `lib/ethdev/` - Ethernet device API
- `lib/mbuf/` - Packet buffer management
- `lib/mempool/` - Memory pool management
- `lib/ring/` - Lock-free ring buffers
- `lib/hash/` - Hash table implementations
- `lib/lpm/` - Longest Prefix Match routing

### Applications
- `app/testpmd/` - Primary testing application for packet forwarding
- `app/test/` - Unit test framework and test suites

## Development Workflow

### Code Style
DPDK follows specific coding standards enforced by `devtools/checkpatches.sh`. The script uses checkpatch.pl with DPDK-specific rules and checks for:
- Line length (max 100 characters)
- Coding style compliance
- Documentation requirements
- ABI compatibility

### Documentation
- `doc/guides/` - Comprehensive documentation in reStructuredText format
- Built using Sphinx: `ninja doc` from build directory
- API documentation auto-generated from code comments

### Testing Framework
- Unit tests in `app/test/`
- DTS (DPDK Test Suite) in `dts/` directory using Python
- Performance and functional test applications in `app/test-*`

## Symbol Management

### Exported Symbol Analysis
DPDK libraries export symbols using specific macros:
- `RTE_EXPORT_SYMBOL()` - Stable public API
- `RTE_EXPORT_EXPERIMENTAL_SYMBOL()` - Experimental API, subject to change
- `RTE_EXPORT_INTERNAL_SYMBOL()` - Internal DPDK use only

Version map files are generated by `buildtools/gen-version-map.py` which scans source code for these export macros.
